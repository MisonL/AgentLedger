1. 先做“治理刚需”，再做“高阶分析”。
2. 能复用现有能力的优先（审计、预算、告警链路复用）。
3. 每项能力都要可验收、可审计、可回滚。

## 4. 每项能力的落地定义

## 4.1 多区域与数据主权策略（P0）

目标：满足跨区域合规与数据驻留要求。

关键能力：

1. 数据驻留策略（租户/组织级）。
2. 跨区传输开关与审批。
3. 区域级归档存储策略。
4. 区域级密钥管理映射（KMS）。

验收标准：

1. 可按租户强制数据驻留在指定区域。
2. 非授权跨区访问被拒绝并记录审计。

## 4.2 Prompt/规则资产中心（P0）

目标：让提示词、系统规则、策略模板“可管可审”。

关键能力：

1. 资产版本管理（Draft/Published/Deprecated）。
2. 审批发布流（至少双人审批可选）。
3. 变更对比与回滚。
4. 适用范围（组织/项目/客户端）。

验收标准：

1. 任意规则变更都可追溯到责任人和审批记录。
2. 支持一键回滚到历史版本。

## 4.3 MCP 工具治理（P0）

目标：规范工具调用，降低误用风险。

关键能力：

1. MCP 工具白名单/黑名单。
2. 风险分级（低/中/高）。
3. 高风险工具审批或二次确认。
4. 工具调用审计与告警。

验收标准：

1. 黑名单工具无法被调用。
2. 高风险工具调用有完整审批与审计链路。

## 4.4 告警编排中心（P0）

目标：统一告警生命周期，减少噪音。

关键能力：

1. 去重、抑制、合并、升级策略。
2. 通知路由（企微/钉钉/飞书/邮件/Webhook）。
3. 告警 ACK/SLA 超时升级。
4. 与工单系统联动（后续）。

验收标准：

1. 同类告警去重率可配置并可观测。
2. 严重告警可自动升级到指定责任人。

## 4.5 合规取证包（P1）

目标：满足审计/稽核/取证需要。

关键能力：

1. 时间窗证据导出。
2. 事件哈希链 + 签名摘要。
3. 审计日志绑定。
4. 校验工具（verify 命令）。

验收标准：

1. 取证包可独立验证完整性和未篡改性。

## 4.6 平台开放能力（P1）

目标：让平台可被企业系统编排调用。

当前代码基线：已落地 OpenAPI、Webhook、API Key 生命周期管理与 8 语言 SDK 生成/校验链路；后续重点放在更多能力开放与分发细节增强。

关键能力：

1. OpenAPI（鉴权、限流、审计）。
2. Webhook（签名校验、重放保护）。
3. SDK（TypeScript/Python/Go/Java/C#/PHP/Ruby/Swift）。
4. API Key 生命周期管理。

验收标准：

1. 第三方系统可稳定调用核心查询与治理接口。

## 4.7 质量评估体系（P1）

目标：从“只看成本”升级到“成本 + 质量”。

当前代码基线：已落地质量评估写入、日报、质量-成本相关性、项目趋势、评分卡与首版控制台查询工作台；后续重点放在更细粒度指标和项目级治理联动。

关键能力：

1. 质量指标定义：通过率、回滚率、缺陷率、修复时长。
2. 质量与成本关联分析。
3. 团队/项目质量得分卡。

验收标准：

1. 仪表盘可按项目显示质量趋势并与成本联动。

## 4.8 会话回放与复现实验室（P2）

目标：复现问题、验证策略、支持回归。

当前代码基线：已落地 replay datasets/runs/diffs/artifacts/download 的 API、SDK 与控制台首版工作台；后续重点放在真实执行链、样本模型与差异分发能力。

关键能力：

1. 会话回放（脱敏后）。
2. 同输入重放对比（模型/策略变更前后）。
3. 回归基线保存。

验收标准：

1. 可对指定历史会话执行一次可重放实验并输出差异报告。
