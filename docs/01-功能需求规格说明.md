# AgentLedger 功能需求规格说明（FRS）

- 文档版本：v1.2
- 更新时间：2026-03-07
- 关联文档：`00-项目愿景与范围.md`

## 1. 需求总览

AgentLedger v1 的目标是交付“可用于生产的多源会话账本系统”，覆盖：

1. 多来源接入（本地 + SSH 远程，支持实时读取与可选同步缓存）。
2. 多客户端统一解析（主流 AI CLI + AI IDE）。
3. 多维度统计（daily/monthly/session/model）。
4. 可视化浏览检索与预算限额监控。

## 1.1 文档适用边界（2026-03-07 补充）

1. 本文档定义的是 `V1 基础账本能力`，重点覆盖 source、session、usage、pricing、budget 等基础产品面。
2. 本文档不展开 `数据主权 / Rule Hub / MCP / 合规取证 / OpenAPI / 质量评估 / 回放实验室` 等 V1.1+ 企业增强能力，这些内容以 `docs/11-企业级增强功能规划.md` 为准。
3. 当前仓库已经实现的企业增强能力，请以 `docs/15-核心功能清单.md` 与 `README.md` 为准，不应再用本文档的早期范围描述否定现有实现。
4. 本文档第 2 章里“v1 默认单组织单租户”的表述，仅代表最初基础需求假设，不代表当前主干代码的最终能力边界。

## 2. 角色与权限

1. `Owner`：管理数据源、定价、预算策略、导出全量数据。
2. `Member`：浏览会话与统计，不可改系统级配置。
3. `ReadOnly`：仅查看报表和会话摘要。

> v1 默认单组织单租户，角色先实现为本地配置权限。

## 4. 非功能需求（NFR）

## 4.1 性能

1. 统计接口（最近 7 天、P0 客户端）P95 < 2.5s。
2. 会话搜索接口 P95 < 1.5s（10 万条标准化事件量级）。
3. 远程 SSH 拉取在网络正常下首包 < 2s。
4. 远程增量同步（近 24h）单源 P95 < 60s（可选能力开启时）。

## 4.2 可靠性

1. 解析任务失败可重试（指数退避，最多 3 次）。
2. 单源故障不影响其他源服务。
3. 关键统计任务具备幂等性。
4. 远程主机不可达时，系统可降级读取最近同步缓存。
5. 客户端格式升级时，未识别字段不应导致全量任务失败。

## 4.3 安全

1. 只读策略：禁写源目录。
2. SSH 使用密钥认证与主机指纹校验。
3. 敏感字段脱敏存储（token、密钥路径、用户标识）。
4. 全量操作审计可追溯。

## 4.4 可维护性

1. 连接器插件化（新增工具不改核心域模型）。
2. 解析器单元测试覆盖率目标 >= 80%。
3. 核心统计口径具备快照测试基线。
4. 支持矩阵按版本维护（P0/P1/P2），每次发布同步更新。
5. 三平台发布件与安装文档同步维护（Linux/macOS/Windows）。

## 5. 业务规则

1. 远程模式默认 `realtime`，可按 Source 开启 `sync` 或 `hybrid`。
2. 同步能力仅支持单向只读缓存，不允许回写远程源。
3. 客户端支持范围遵循支持矩阵（`09-主流AI客户端支持矩阵.md`）。
4. 成本计算优先级：`event.cost` > `pricing_catalog_estimate`。
5. 重复事件去重键：`source + session_id + event_id + timestamp`。
6. 时间统一存储为 UTC，展示按用户时区转换。

## 6. 用户故事（节选）

1. 作为开发者，我希望按“客户端类型+工具+模型+日期”查看 token 消耗，快速发现异常。
2. 作为负责人，我希望查看团队周成本趋势，判断预算是否超支。
3. 作为运维，我希望远程在线时走 SSH 实时读取，离线时自动回退到同步缓存。
4. 作为分析者，我希望点击某天峰值直接下钻到具体会话。

## 7. MVP 验收清单（DoD）

1. P0 客户端解析正确率抽样 >= 99%。
2. 支持 >= 5 台远程主机并发实时拉取，并支持单源同步缓存任务。
3. CLI + IDE 的 P0 客户端全部可用。
4. Daily/Monthly/Session/Model 四类报表全部可用。
5. 会话搜索支持关键词 + 过滤器组合。
6. 远程离线时可回退缓存查询，并展示缓存时间戳。
7. 预算阈值告警可触发并可见，且 callback 解冻动作存在预算绑定校验。
8. 安全审计日志覆盖关键管理操作。

## 8. 依赖与前置条件

1. 远程主机可通过 SSH 密钥登录。
2. 远程主机上目标路径可读。
3. 本地运行环境采用 Bun（优先）和现代浏览器。
4. 服务端与 Client 部署环境需覆盖 Linux/macOS/Windows 三平台。
