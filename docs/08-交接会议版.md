# AgentLedger 交接会议版（30 分钟）

- 文档版本：v1.0
- 日期：2026-03-01
- 使用场景：项目交接会 / 启动会 / 周会复盘

## 1. 会议目标

1. 统一项目边界与约束。
2. 明确实现阶段负责人、里程碑与时间线。
3. 确认首周行动项与验收标准。

## 2. 参会角色与职责

1. 老板（项目 Owner）
   - 决策范围与优先级。
   - 确认资源与时间。
2. 技术负责人（接手 TL）
   - 拆解实施计划。
   - 评估风险与技术债。
3. 后端负责人
   - 连接器、解析器、统计引擎。
4. 前端负责人
   - Dashboard/Sessions/Sources/Budgets 页面。
5. 测试负责人
   - 金样本、回归、E2E 验收。
6. 运维/平台负责人
   - SSH 远程主机连通性与安全策略。

## 3. 会议前准备（T-1 天）

1. 阅读文档：`docs/00` 到 `docs/11`（重点 01/02/04/09/10/11）。
2. 准备样本：CLI + IDE（含国产 Qwen/TRAE）日志样本各 >= 10 份。
3. 准备环境：至少 1 台可 SSH 的远程测试主机（只读账号）。
4. 准备三平台验证环境：Linux/macOS/Windows（Server+Client）。
5. 准备问题：记录阻塞点与依赖项。

## 4. 30 分钟议程

## 4.1 0~5 分钟：项目现状

1. 当前阶段：前置设计完成，未进入编码。
2. 交付资产：需求、架构、数据、UI/UX、计划、交接文档。
3. 目标：本周进入实现阶段。

## 4.2 5~12 分钟：范围与约束确认

1. 功能目标：多源会话 + token/cost 统计 + 可视化管理。
2. 核心约束：
   - 远程支持 `realtime` / `sync` / `hybrid`，默认 `realtime`。
   - 同步仅允许单向只读缓存，不允许回写远程源。
   - 系统默认只读采集。
3. MVP 范围冻结（避免临时扩 scope）。

## 4.3 12~20 分钟：技术方案与分工

1. 后端方案：Bun + Hono + SQLite。
2. 前端方案：React + TanStack Query。
3. 分工确认：
   - 后端：Source/Parser/Normalizer/API。
   - 前端：Dashboard/Sessions/Sources/Budgets。
   - 测试：金样本 + E2E 基线。

## 4.4 20~27 分钟：里程碑与风险

1. M1（第 1 周）：工程底座。
2. M2（第 2~3 周）：采集解析闭环。
3. M3（第 4 周）：统计查询与核心 UI。
4. 风险复核：日志格式变化、SSH 抖动、成本口径争议。

## 4.5 27~30 分钟：会后行动项确认

1. 明确 owner、截止时间、验收口径。
2. 明确下次同步会议时间。
3. 会后 24h 内发出会议纪要。

## 5. 决策清单（会议中逐项确认）

1. 是否确认 MVP 范围不变。
2. 是否确认“默认实时 + 可选同步缓存”策略。
3. 是否确认首周资源（人力/主机/样本）到位。
4. 是否确认验收指标（解析正确率、统计误差、P95）。

## 7. 首周任务建议（可直接执行）

1. Day 1：建仓与脚手架、数据库迁移、日志规范。
2. Day 2：LocalConnector 与样本落库闭环。

5. Day 5：Dashboard（含热力图）/Sessions 联调 + 周末演示。

## 9. 会议主持词（简版）

1. 开场：今天目标是完成项目交接并锁定首周执行计划。
2. 范围确认：请先确认 MVP 与核心约束，避免后续返工。
3. 分工确认：逐项确认 owner、截止、验收标准。

1. 每条 P0 任务都有明确 owner。
2. 每条任务都有截止时间与验收口径。
3. 首周样本与测试主机已落实。
4. 下次同步会时间已锁定。
