# AgentLedger 项目愿景与范围

- 文档版本：v1.0
- 更新时间：2026-03-01
- 负责人：老板（产品 Owner）、员工（方案设计）

## 1. 项目背景

当前 Codex、Claude Code、Gemini CLI 等 AI CLI 的会话数据分散在不同目录与机器，导致：

1. 会话难以统一检索（跨工具、跨项目、跨主机）。
2. token/cost 统计口径不一致（按天/月/会话聚合困难）。
3. 远程主机会话不可见，复盘与审计成本高。
4. 可视化管理与限额监控缺失，难做团队治理。

## 2. 产品定位

**AgentLedger** 定位为一个“多 AI CLI 的会话账本与分析平台”，融合以下能力：

1. `tokscale`：多工具 token/cost 汇总。
2. `cass`：跨主机（本地 + 远程）会话聚合。
3. `ccusage`：按天/月/会话统计模型。
4. `agent-sessions`：会话浏览、检索、限额可视化。

## 3. 产品目标

### 3.1 业务目标

1. 在单一界面统一管理主流 AI CLI 与 AI IDE 的历史会话。
2. 提供可信的 token/cost 统计与趋势分析。
3. 支持远程主机会话 SSH 实时读取，并提供可选同步缓存（离线可查）。
4. 支持个人与小团队的日常使用治理（预算、限额、复盘）。

### 3.2 成功指标（上线后 60 天）

1. 多源接入成功率 >= 98%。
2. 每日统计任务成功率 >= 99%。
3. 关键查询（最近 7 天 + P0 客户端）P95 响应 < 2.5s。
4. 统计结果与源日志抽样误差 < 1%。

## 4. 范围定义

## 4.1 V1 必做范围（MVP）

1. 数据源管理：本地源 + SSH 远程源。
2. 首批客户端支持（CLI + IDE）：
   - CLI：Codex、Claude Code、Gemini CLI、Aider、OpenCode、Qwen Code。
   - IDE：Cursor、VS Code（Cline/Roo/Continue/Copilot Chat）、TRAE IDE。
3. 统一会话浏览：筛选、搜索、按来源追溯。
4. 统计看板：daily/monthly/session/model 维度 token/cost。
5. 限额跟踪：按日/周/月预算与消耗进度。
6. 导出能力：CSV/JSON 报表导出。
7. 审计能力：保留原始来源引用（主机、路径、会话 ID）。

## 4.2 V1 不做范围（Non-Goals）

1. 不做远程会话“回写源端”的双向同步（仅支持单向只读同步到本地缓存）。
2. 不做会话内容写回或“继续远程会话”的控制操作。
3. 不做“所有客户端一次性全覆盖”，采用分批接入（见支持矩阵）。
4. 不做企业级 SSO/多租户隔离（仅单租户团队版预留）。
5. 不做移动端 App（仅桌面 Web 响应式）。

## 5. 关键约束

1. **远程策略**：默认 SSH 实时读取；可选开启只读同步缓存（单向、可关闭）。
2. **权限策略**：默认只读，禁止对源目录执行写入或删除操作。
3. **工具链策略**：后端与脚本优先 Bun + TypeScript。
4. **兼容策略**：服务端与 Client 均支持三大系统平台（Linux/macOS/Windows）。
5. **覆盖策略**：客户端支持采用“P0/P1/P2 分层”持续扩展，不在核心域写死三工具枚举。

## 6. 目标用户

1. 个人开发者：需要看清 token 消耗、复盘历史会话。
2. 技术负责人：需要跨项目看团队 AI 成本与效率。
3. 平台/效能工程师：需要规范化审计和预算治理。

## 7. 核心使用场景

1. 我想知道本周所有 AI CLI + AI IDE 总共用了多少 token、花费多少。
2. 我想查某个问题在远程构建机上的历史会话里是否出现过。
3. 我想按模型查看消耗占比，找到成本异常峰值。
4. 我想给团队设置预算阈值并在接近上限时收到提示。
5. 我想在远程主机离线时，仍可通过最近一次同步缓存查询历史会话。

## 8. 术语表

1. `Source`：会话来源（本地或远程主机）。
2. `Session`：单次 AI CLI 会话。
3. `Usage`：token 使用量（input/output/cache/reasoning）。
4. `Cost`：基于定价模型估算或原始日志记录的成本。
5. `Realtime SSH Pull`：通过 SSH 即时执行读取。
6. `Readonly Sync Cache`：将远程会话单向只读同步到本地缓存，用于离线查询与降级。
7. `Client Connector`：面向具体 AI CLI/IDE 的适配器插件。
