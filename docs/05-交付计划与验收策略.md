# AgentLedger 交付计划与验收策略

- 文档版本：v2.1
- 更新时间：2026-03-03
- 关联文档：`01-功能需求规格说明.md`、`02-软件架构设计.md`、`13-环境变量参考.md`

## 1. 交付目标

在不牺牲可追溯性与可运维性的前提下，分阶段交付可运行、可验证、可审计的企业级 AI 使用治理平台。

## 2. 里程碑规划

| 里程碑 | 周期 | 交付重点 | 退出标准（DoD） |
| --- | --- | --- | --- |
| M1 工程底座 | W1 | Monorepo、基础 API/Web、脚本体系 | `lint/test/build` 可在本地与 CI 稳定执行 |
| M2 采集解析 | W2-W3 | P0 Connector + 统一事件模型 + 解析链路 | P0 客户端最小采集闭环打通，失败重试可观测 |
| M3 统计检索 | W4 | usage 聚合、热力图、sessions 检索 | usage 与 sessions 基础 API/UI 可用，关键路径有测试 |
| M4 治理回调 | W5 | budgets、alerts、integration callback | governance -> integration -> control-plane 链路通过脚本化回归 |
| M5 稳定发布 | W6 | 跨平台构建、文档完备、发布检查 | 多架构构建通过（含 `darwin-arm64`），三平台最小运行冒烟通过 |

## 3. 验收策略

### 3.1 分层测试

1. 单元测试：解析逻辑、预算规则、回调安全校验。
2. 集成测试：source/sessions/pricing/alerts 等 API 与存储协同。
3. 联调测试：callback 链路、支持矩阵与配置绑定一致性。
4. 发布回归：核心页面可访问、核心脚本可执行、关键接口可响应。

### 3.2 发布门槛

1. 功能门槛：MVP 范围内接口与页面达到“最小可用”。
2. 质量门槛：`lint`、`test`、`build` 全绿。
3. 配置门槛：回调链路变量与代码绑定一致，禁止文档漂移。
4. 文档门槛：README 与 `docs/05`、`docs/09`、`docs/13`、`docs/15` 同步更新。

## 4. 门禁脚本映射

| 目标 | 命令 | 脚本 |
| --- | --- | --- |
| 类型检查 | `bun run lint` | `scripts/lint.sh`、`scripts/ts-check.sh` |
| 全量测试 | `bun run test` | `scripts/test.sh` |
| Go 测试 | `bun run test:go` | `scripts/go-test-all.sh` |
| 构建检查 | `bun run build` | `scripts/build.sh` |
| 覆盖率门禁 | `bun run test:coverage` | `scripts/test-coverage.sh`、`scripts/check-coverage-threshold.sh` |
| 客户端支持矩阵一致性 | `bun run check:support-matrix` | `scripts/check-support-matrix.ts` |
| 回调绑定一致性 | `bun run check:callback-stream-binding` | `scripts/check-callback-stream-binding.sh` |
| 回调链路定向回归 | `bun run test:callback-chain-targeted` | `scripts/test-callback-chain-targeted.sh` |
| 回调链路闭环冒烟 | `bun run test:e2e-governance-callback-chain` | `scripts/e2e-governance-callback-chain.sh` |
| FR-505 三平台最小冒烟 | `bun run smoke:fr505` | `scripts/smoke-fr505-cross-platform.sh`、`scripts/wait-http-json.ts`、`scripts/mock-ingestion-smoke.ts` |
| Agent 多架构构建验收 | `bun run build:agent-cross && bun run verify:agent-cross` | `scripts/build-agent-cross.sh`、`scripts/verify-agent-cross-artifacts.sh` |

### 4.1 Coverage 门禁阈值

1. Go 核心包阈值：
   - `services/ingestion-gateway` 行覆盖率 `>= 70%`
   - `services/puller` 行覆盖率 `>= 70%`
   - `services/integration` 行覆盖率 `>= 75%`
2. `apps/control-plane` 门槛：`All files` 行覆盖率 `>= 80%`。
3. 统一入口：`bun run test:coverage`（采集 + 阈值校验）。

## 5. 风险与缓解

| 风险 | 影响 | 缓解策略 |
| --- | --- | --- |
| 客户端日志格式漂移 | 采集与解析失败 | 版本探测 + 兼容解析 + 回退策略 |
| 回调配置不一致 | 告警闭环中断 | 固化变量命名 + 脚本化一致性校验 |
| 统计口径争议 | 报表可信度下降 | 原始值与估算值并行展示，保留计算口径 |
| 文档滞后代码 | 联调误判、上线风险 | 文档与脚本联动校验，评审清单强制核对 |
