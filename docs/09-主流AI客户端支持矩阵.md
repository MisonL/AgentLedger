# AgentLedger 主流 AI 客户端支持矩阵

- 文档版本：v2.1
- 更新时间：2026-03-03
- 目标：定义客户端接入优先级，并与 puller connector 实现保持可校验的一致性。

## 1. 优先级定义

1. `P0`：首发必达，必须进入 MVP 采集范围。
2. `P1`：首发后优先补齐，按业务价值排序接入。
3. `P2`：预研与生态驱动，按资源窗口推进。

## 4. 与门禁脚本的一致性规则

`scripts/check-support-matrix.ts` 会读取本文档中的 P0/P1 客户端，并对 `services/puller/connectors.go` 执行双向严格校验（缺失/冗余均失败），同时校验 parser 入口是否存在。规则如下：

1. P0/P1 客户端名称必须保留关键识别词，并映射到既有 connector：
   - `codex` -> `codex`
   - `claude` -> `claude`
   - `gemini` -> `gemini`
   - `aider` -> `aider`
   - `opencode` -> `opencode`
   - `qwen` -> `qwen-code`
   - `kimi` -> `kimi-cli`
   - `cursor` -> `cursor`
   - `vs code` -> `vscode`
   - `trae ide` -> `trae-ide`
   - `trae cli` -> `trae-cli`
   - `codebuddy code cli` -> `codebuddy-cli`
   - `windsurf` -> `windsurf`
   - `lingma` -> `lingma`
   - `codebuddy ide` -> `codebuddy-ide`
   - `zed` -> `zed`
2. 若新增或调整 P0/P1 客户端，必须同步更新：
   - 本文档矩阵条目
   - `scripts/check-support-matrix.ts` 的映射规则
   - `services/puller/connectors.go` 的 connector 声明
3. PR 验证必须包含：`bun run check:support-matrix`，并通过以下门禁：
   - docs/09 P0/P1 与 puller connector 双向严格一致
   - `featureConnector.Parse` 存在 parser 入口调用
   - `parseWithConnector` 同时存在调度入口与回退入口
