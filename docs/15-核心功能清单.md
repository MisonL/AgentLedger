# AgentLedger 核心功能清单

- 文档版本：v1.11
- 更新时间：2026-03-04
- 目标：定义当前可交付核心能力边界，作为需求评审、联调与验收基线。

## 1. 采集与接入

| 功能项 | 说明 | 入口/实现 |
| --- | --- | --- |
| 多源接入模型 | 统一抽象 CLI/IDE/source 的接入配置 | `apps/control-plane` sources API |
| agent 自动采集（新增） | `agent collect` 支持按 docs/09 P0/P1 客户端矩阵自动采集本机会话（CLI + IDE）；显式 `--tool=<client-key>` 与 `--tool=auto` 均可解析。典型命令：`agent collect`、`agent collect --help`。 | `clients/agent` |
| Source 生命周期 | source 创建、查询、删除、多租户隔离 | `/api/v1/sources*` |
| 连通性测试 | 支持 sourceId 与临时配置两种模式 | `/api/v1/sources/:id/test-connection`、`/api/v1/sources/test-connection` |
| 同步任务编排 | 创建、查询、取消同步任务 | `/api/v1/sources/:id/sync-jobs`、`/api/v1/sync-jobs/:id/cancel` |
| 同步失败重试 | `sessions/search` 触发 puller `sync-now` 失败时支持指数退避重试（最大尝试、基线退避、退避上限可配置），超限后回退缓存并标注原因；puller 侧仅 `sync/hybrid` 模式启用任务失败重试，`realtime` 失败直接落库为 failed。 | `/api/v1/sessions/search` + `PULLER_SYNC_RETRY_*` + `services/puller` |
| SSH 目录多文件拉取（新增） | 当 `source.path` 指向目录时，按稳定排序拉取远端目录下受支持文件（`.json/.jsonl/.ndjson/.md/.log/.txt`）；指向文件时保持原有单文件兼容。 | `services/puller` SSH source 执行链路 |
| 来源健康与解析失败追踪 | 提供单源健康指标与最近解析失败明细 | `/api/v1/sources/:id/health`、`/api/v1/sources/:id/parse-failures` |

## 2. 会话与事件

| 功能项 | 说明 | 入口/实现 |
| --- | --- | --- |
| 会话列表检索 | 按关键词与多维过滤搜索会话 | `/api/v1/sessions/search` |

| 事件时间线 | 分页返回会话事件列表 | `/api/v1/sessions/:id/events` |

## 7. 审计与安全

| 功能项 | 说明 | 入口/实现 |
| --- | --- | --- |
| 审计日志 | 记录关键治理与系统动作 | `/api/v1/audits` |
| 审计导出（新增） | 支持按过滤条件导出审计日志（JSON/CSV）并记录导出审计 | `/api/v1/audits/export` |
| identity 绑定（新增） | 支持租户/组织/成员与 `device/agent/source-binding` 绑定闭环：创建租户自动绑定创建者为 owner，成员绑定支持 `tenantRole + organizationId + orgRole`；`device/agent/source-binding` 支持 list/create/delete；跨租户访问返回 403；非 owner/maintainer 写操作返回 403。 | `/api/v1/tenants*`、`/api/v1/tenants/:tenantId/(orgs|organizations|members|devices|agents|source-bindings)` + `apps/control-plane/src/routes/identity.ts` + `apps/control-plane/test/api.test.ts` |
| 登录提供方发现（新增） | `auth providers` 统一返回本地登录与外部 OAuth/OIDC/SSO 配置（支持通过环境变量动态注入 provider 列表，供前端登录页渲染）；当 `AUTH_DISABLE_LOCAL_LOGIN=true` 时，除隐藏本地 provider 外还会硬拦截本地 `register/login` 接口。 | `GET /api/v1/auth/providers` + `AUTH_DISABLE_LOCAL_LOGIN` + `AUTH_EXTERNAL_PROVIDERS_JSON` |
| OAuth/OIDC 授权码交换登录（新增） | 服务端按 provider 配置执行 code -> token（可选 userInfo）并签发平台会话；能力边界：仅处理交换与会话签发，不托管上游 IdP 登录页/MFA，且不向前端透出 `clientSecret`。 | `POST /api/v1/auth/external/exchange` + `AUTH_EXTERNAL_PROVIDERS_JSON`(`tokenUrl/userInfoUrl/clientId/clientSecret/scopes`) |
| 外部断言登录（新增） | 支持企业 IdP/SSO 网关通过签名断言换取平台会话，具备时间窗校验、nonce 防重放，并记录成功/失败审计。 | `POST /api/v1/auth/external/login` + `AUTH_EXTERNAL_ASSERTION_SECRET` |
| 多租户隔离 | 资源与审计按 tenant 隔离 | control-plane repository + tests |
| 回调防重放 | callback_id 幂等处理与跨租户隔离 | integration-callback routes |

| 页面 | 最小可用能力 |

| Pricing | 定价目录读取与更新 |

## 9. 质量工程基线

1. 本地门禁：`bun run lint`、`bun run test`、`bun run build`。
2. 一致性门禁：`bun run check:support-matrix`（P0/P1 双向严格 + parser 入口校验）、`bun run check:callback-stream-binding`。
3. 回调链路回归：`bun run test:callback-chain-targeted`、`bun run test:e2e-governance-callback-chain`。
4. coverage 门禁：`bun run test:coverage`，当前阈值为 ingestion-gateway>=70%、puller>=70%、integration>=75%、control-plane(All files lines)>=80%。
5. 定向验收：`go test ./services/analytics -run TestBuildWeeklySummaryResponse -count=1`、`bash ./scripts/test-identity-targeted.sh`、`bun test apps/control-plane/test/api.test.ts -t "POST /api/v1/integrations/callbacks/alerts 支持全 action 与 callback_id 幂等"`。
