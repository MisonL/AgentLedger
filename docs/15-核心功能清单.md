# AgentLedger 核心功能清单

- 文档版本：v1.20
- 更新时间：2026-03-07
- 目标：定义当前可交付核心能力边界，作为需求评审、联调与验收基线。

## 1. 采集与接入

| 功能项 | 说明 | 入口/实现 |
| --- | --- | --- |
| 多源接入模型 | 统一抽象 CLI/IDE/source 的接入配置 | `apps/control-plane` sources API |
| agent 自动采集（新增） | `agent collect` 支持按 docs/09 P0/P1 客户端矩阵自动采集本机会话（CLI + IDE）；显式 `--tool=<client-key>` 与 `--tool=auto` 均可解析。典型命令：`agent collect`、`agent collect --help`。 | `clients/agent` |
| Source 生命周期 | source 创建、查询、删除、多租户隔离 | `/api/v1/sources*` |
| 连通性测试 | 支持 sourceId 与临时配置两种模式 | `/api/v1/sources/:id/test-connection`、`/api/v1/sources/test-connection` |
| 同步任务编排 | 创建、查询、取消同步任务 | `/api/v1/sources/:id/sync-jobs`、`/api/v1/sync-jobs/:id/cancel` |
| 同步失败重试 | `sessions/search` 触发 puller `sync-now` 失败时支持指数退避重试（最大尝试、基线退避、退避上限可配置），超限后回退缓存并标注原因；puller 侧仅 `sync/hybrid` 模式启用任务失败重试，`realtime` 失败直接落库为 failed。 | `/api/v1/sessions/search` + `PULLER_SYNC_RETRY_*` + `services/puller` |
| SSH 目录多文件拉取（新增） | 当 `source.path` 指向目录时，按稳定排序拉取远端目录下受支持文件（`.json/.jsonl/.ndjson/.md/.log/.txt`）；指向文件时保持原有单文件兼容。 | `services/puller` SSH source 执行链路 |
| 来源健康与解析失败追踪 | 提供单源健康指标与最近解析失败明细 | `/api/v1/sources/:id/health`、`/api/v1/sources/:id/parse-failures` |

## 9. 质量工程基线

1. 本地门禁：`bun run lint`、`bun run test`、`bun run build`。
2. 一致性门禁：`bun run check:support-matrix`（P0/P1 双向严格 + parser 入口校验）、`bun run check:callback-stream-binding`。
3. SDK 门禁：`bun run sdk:check`（只读校验 OpenAPI 与 SDK 元数据一致性、8 语言 operation 全覆盖、入口方法完整性与 `SHA256SUMS` 一致性；发布门禁默认执行 `sdk:verify + sdk:test`，需要只读定位时再用 `sdk:check`）。
4. SDK 一键构建：`bun run sdk:build`（生成 -> 校验 -> 覆盖测试 -> 产物打包到 `dist/sdk`，附 `SHA256SUMS.txt`）。
5. integration / callback 链路回归：`bun run test:e2e-integration-routing`、`bun run test:callback-chain-targeted`、`bun run test:e2e-governance-callback-chain`、`GOV_E2E_DATABASE_URL=... bun run test:e2e-governance-integration-downstream`（真实覆盖 governance `alert/weekly` 发布到 integration 外部下游）。
6. coverage 门禁：`bun run test:coverage`，当前阈值为 ingestion-gateway>=70%、puller>=70%、integration>=75%、control-plane(All files lines)>=80%。
7. 定向验收：`go test ./services/analytics -run TestBuildWeeklySummaryResponse -count=1`、`bash ./scripts/test-identity-targeted.sh`、`bun test apps/control-plane/test/api.test.ts -t "POST /api/v1/integrations/callbacks/alerts 支持全 action 与 callback_id 幂等"`。
8. 治理链真实 E2E：`GOV_E2E_DATABASE_URL=... bun run test:e2e-governance-routing`，验证 `fallback / dedupe / suppressed / fail-open / weekly`；该脚本只覆盖 governance 服务在真实 PostgreSQL + 嵌入式 JetStream 上的发布与落库。
9. 取证校验：`bun run evidence:verify -- --file <bundle.json>`（需提供 `--signing-key` 或环境变量 `EVIDENCE_BUNDLE_SIGNING_KEY`）。
