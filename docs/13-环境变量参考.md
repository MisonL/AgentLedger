# AgentLedger 环境变量参考

- 文档版本：v2.0
- 更新时间：2026-03-03
- 适用范围：`services/ingestion-gateway`、`services/integration`、`apps/control-plane`、`clients/agent`

## 1. 使用约定

1. 所有变量名采用全大写下划线。
2. 未特别说明时，默认值仅用于本地开发，不代表生产建议值。
3. 涉及回调鉴权的密钥变量必须在 integration 与 control-plane 两端保持一致。

## 2. ingestion-gateway 关键变量

| 变量名 | 默认值 | 必填条件 | 说明 |
| --- | --- | --- | --- |
| `NATS_URL` | `nats://127.0.0.1:4222` | 否 | NATS 连接地址。 |
| `HTTP_ADDR` | `:8081` | 否 | HTTP 监听地址。 |
| `GRPC_ADDR` | `:9091` | 否 | gRPC 监听地址。 |
| `GRPC_TLS_ENABLED` | `true` | 否 | 是否启用 gRPC TLS。 |
| `GRPC_TLS_CERT_FILE` | 空 | `GRPC_TLS_ENABLED=true` 时必填 | gRPC 证书（PEM）。 |
| `GRPC_TLS_KEY_FILE` | 空 | `GRPC_TLS_ENABLED=true` 时必填 | gRPC 私钥（PEM）。 |
| `GRPC_MTLS_ENABLED` | `false` | 否 | 是否启用 mTLS。 |
| `GRPC_TLS_CLIENT_CA_FILE` | 空 | `GRPC_MTLS_ENABLED=true` 时必填 | 客户端 CA 证书。 |
| `OIDC_ISSUER` | 空 | 启动必填 | OIDC issuer 校验值。 |
| `OIDC_AUDIENCE` | 空 | 启动必填 | OIDC audience 校验值。 |
| `OIDC_JWKS_URI` | 空 | 启动必填 | JWKS 地址。 |
| `OIDC_CLIENT_ID` | 空 | 否（见说明） | 设备码代理默认 `client_id`。 |
| `OIDC_DEVICE_AUTH_ENDPOINT` | 空 | 设备码 start 接口需要 | 未配置时 `/v1/oidc/device/start` 返回 `503`。 |
| `OIDC_TOKEN_ENDPOINT` | 空 | 设备码 poll 接口需要 | 未配置时 `/v1/oidc/device/poll` 返回 `503`。 |

## 3. integration 关键变量

### 3.1 告警分发

| 变量名 | 默认值 | 必填条件 | 说明 |
| --- | --- | --- | --- |
| `INTEGRATION_STREAM` | `GOVERNANCE_ALERTS` | 否 | 告警 stream 名称。 |
| `INTEGRATION_SUBJECT` | `governance.alerts` | 否 | 告警订阅主题。 |
| `INTEGRATION_DURABLE` | `INTEGRATION_ALERTS_DISPATCHER` | 否 | 告警 durable consumer。 |
| `INTEGRATION_WEEKLY_STREAM` | `GOVERNANCE_REPORTS_WEEKLY` | 否 | 周报 stream。 |
| `INTEGRATION_WEEKLY_SUBJECT` | `governance.reports.weekly` | 否 | 周报主题。 |
| `INTEGRATION_WEEKLY_DURABLE` | `INTEGRATION_WEEKLY_REPORT_DISPATCHER` | 否 | 周报 durable consumer。 |
| `INTEGRATION_DLQ_SUBJECT` | `integration.dispatch` | 否 | 分发失败 DLQ 主题。 |
| `INTEGRATION_CHANNELS` | `webhook` | 否 | 启用通道列表：`webhook,wecom,dingtalk,feishu`；`webhook` 原样转发事件，机器人通道发送 `text` 模板消息。 |
| `INTEGRATION_WEBHOOK_URL` | 空 | 启用 `webhook` 时必填 | Webhook 地址（`http/https`）。 |
| `INTEGRATION_WECOM_WEBHOOK_URL` | 空 | 启用 `wecom` 时必填 | 企业微信机器人地址。 |
| `INTEGRATION_DINGTALK_WEBHOOK_URL` | 空 | 启用 `dingtalk` 时必填 | 钉钉机器人地址。 |
| `INTEGRATION_FEISHU_WEBHOOK_URL` | 空 | 启用 `feishu` 时必填 | 飞书机器人地址。 |
| `INTEGRATION_ROUTING_MODE` | `broadcast` | 否 | 路由模式：`broadcast`/`severity`。 |
| `INTEGRATION_WEBHOOK_TIMEOUT` | `10s` | 否 | 单次 webhook 超时。 |
| `INTEGRATION_RETRY_MAX` | `5` | 否 | 最大重试次数。 |
| `INTEGRATION_RETRY_BASE_DELAY` | `2s` | 否 | 退避基线。 |
| `INTEGRATION_RETRY_MAX_DELAY` | `60s` | 否 | 退避上限。 |
| `INTEGRATION_CONSUMER_ACK_WAIT` | `90s` | 否 | Consumer ack 等待时间。 |
| `INTEGRATION_DLQ_PUBLISH_TIMEOUT` | `5s` | 否 | DLQ 发布超时。 |

### 3.1.1 integration 通道格式与事件支持

| 通道 | 出站格式 | 支持事件类型 | 说明 |
| --- | --- | --- | --- |
| `webhook` | 原样转发上游事件 JSON（`application/json`） | `alert`、`weekly` | 不改写业务字段，接收端按治理事件 schema 解析。 |
| `wecom` | `text` 模板消息 | `alert`、`weekly` | 发送企业微信机器人文本模板。 |
| `dingtalk` | `text` 模板消息 | `alert`、`weekly` | 发送钉钉机器人文本模板。 |
| `feishu` | `text` 模板消息 | `alert`、`weekly` | 发送飞书机器人文本模板。 |

### 3.2 回调转发链路

| 变量名 | 默认值 | 必填条件 | 说明 |
| --- | --- | --- | --- |
| `INTEGRATION_CALLBACK_STREAM` | `INTEGRATION_CALLBACK_EVENTS` | 否 | 回调事件消费 stream。 |
| `INTEGRATION_CALLBACK_SUBJECT` | `integration.callback.events` | 否 | 回调事件消费 subject。 |
| `INTEGRATION_CALLBACK_TOPIC` | 空 | 否 | `INTEGRATION_CALLBACK_SUBJECT` 的兼容别名；二者同时配置时以 `SUBJECT` 为准。 |
| `INTEGRATION_CALLBACK_DURABLE` | `INTEGRATION_CALLBACK_EVENT_SINK` | 否 | 回调事件 durable consumer。 |
| `CONTROL_PLANE_BASE_URL` | 空 | 启动必填 | control-plane 基础地址。 |
| `INTEGRATION_CALLBACK_PATH` | `/api/v1/integrations/callbacks/alerts` | 否 | 回调转发路径，支持相对路径并自动标准化。 |
| `INTEGRATION_CALLBACK_SECRET` | 空 | 非测试环境启动必填 | integration 写入 `X-Integration-Callback-Secret`，并用于入站 callback 鉴权。 |

integration 入站 endpoint：`POST /v1/callbacks/alerts`。

## 4. control-plane 回调变量

| 变量名 | 默认值 | 必填条件 | 说明 |
| --- | --- | --- | --- |
| `INTEGRATION_CALLBACK_SECRET` | 空 | 生产环境必填 | 校验 `POST /api/v1/integrations/callbacks/alerts` 请求头 `x-integration-callback-secret`。 |

说明：当前实现已不使用 `CONTROL_PLANE_CALLBACK_*` 系列变量。

## 5. DATABASE_URL 依赖说明

1. `services/integration`：当前告警分发与回调转发链路不强依赖 `DATABASE_URL`。
2. `services/ingestion-gateway`：当前不强依赖 `DATABASE_URL`。
3. 强依赖 `DATABASE_URL` 的服务：`services/normalizer`、`services/analytics`、`services/governance`、`services/puller`、`services/archiver`。
4. `apps/control-plane`：未配置 `DATABASE_URL` 时会回退内存模式；生产环境建议显式配置 PostgreSQL。

## 6. 回调链路一致性对照表

| 对照项 | integration 侧 | control-plane 侧 | 校验脚本 |
| --- | --- | --- | --- |
| Stream/Subject/Durable 绑定 | `INTEGRATION_CALLBACK_STREAM` / `INTEGRATION_CALLBACK_SUBJECT`(`TOPIC`) / `INTEGRATION_CALLBACK_DURABLE` | - | `scripts/check-callback-stream-binding.sh` |
| 转发目标 URL | `CONTROL_PLANE_BASE_URL` + `INTEGRATION_CALLBACK_PATH` | 路由 `/api/v1/integrations/callbacks/alerts` | `scripts/test-callback-chain-targeted.sh` |
| 回调鉴权 Secret | header `X-Integration-Callback-Secret` | 校验 `x-integration-callback-secret` | `scripts/test-callback-chain-targeted.sh`、`scripts/e2e-governance-callback-chain.sh` |

建议联调命令：

```bash
bun run check:callback-stream-binding
bun run test:callback-chain-targeted
bun run test:e2e-governance-callback-chain
```
