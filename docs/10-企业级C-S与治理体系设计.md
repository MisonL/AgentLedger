# AgentLedger 企业级 C/S 与治理体系设计

- 文档版本：v1.0
- 更新时间：2026-03-01
- 适用范围：企业私有化部署 / 混合云部署 / SaaS 多租户部署

## 1. 目标与边界

在现有 `SSH 实时读取 + 可选同步` 的能力上，新增企业级 `C/S` 体系：

1. 在目标主机安装 `Agent Client`，持续采集并回传 AI 使用数据。
2. 服务端统一存储、汇总、检索、分析并治理。
3. 建立完整身份体系（组织、账号、设备、客户端绑定）。
4. 支持 OAuth/OIDC/企业 SSO 登录接入。
5. 支持企业办公软件集成（企微、钉钉、飞书）。
6. 支持归档数据落盘到对象存储或服务器本机文件系统（可二选一或混合）。
7. 具备企业级管控与审计能力。

## 2. 总体架构（双采集模式）

```text
采集层（Edge）
├─ SSH Pull（无 Agent）
└─ Agent Push（C/S）

平台层（Control Plane + Data Plane）
├─ API Gateway / Auth Gateway
├─ Ingestion Service（实时接入）
├─ Normalization Service（统一模型）
├─ Session Service（会话检索）
├─ Usage Analytics Service（统计/热力图）
├─ Governance Service（预算/策略/审计）
├─ Identity Service（组织/账号/绑定）
└─ Integration Service（企微/钉钉/飞书/Webhook）

存储层
├─ OLTP（PostgreSQL）
├─ 搜索（OpenSearch/ClickHouse 可选）
├─ 缓存（Redis）
└─ Archive Storage（Object Storage 或 Local Filesystem）
```

## 3. 核心能力域（能力清单）

## 3.1 数据采集能力

1. 双模式采集：`SSH Pull` + `Agent Push`。
2. 主机离线容忍：Agent 本地缓冲（Store-and-Forward）。
3. 多客户端采集：CLI/IDE 插件化适配器。
4. 增量采集：水位、偏移、幂等去重。
5. 数据质量标记：high/medium/low。

## 3.2 数据处理能力

1. 多源事件标准化为 `NormalizedEvent`。
2. 统一 usage/cost 口径与版本化定价。
3. 支持日/月/会话/模型/客户端统计。
4. GitHub 风格热力图聚合（tokens/cost/sessions）。
5. 实时计算 + 离线重算双通道。

## 3.3 检索与分析能力

1. 全局会话检索（全文 + 结构化过滤）。
2. 跨主机/跨客户端/跨项目下钻。
3. 异常检测（突增、异常模型、异常来源）。
4. 报表导出（CSV/JSON/可审计链接）。

## 3.4 管控与合规能力

1. 预算策略（组织/项目/用户/模型维度）。
2. 阈值告警与升级策略（50/80/100%）。
3. 权限模型（RBAC + ABAC）。
4. 完整审计（登录、查询、导出、策略变更）。
5. 数据保留与删除策略（Retention/Legal Hold）。

## 4. C/S 架构设计

## 4.1 Agent Client（部署在目标主机）

### 功能模块

1. `Collector Manager`：管理各客户端采集器。
2. `Source Adapters`：Codex/Claude/Gemini/Qwen/Kimi/TRAE/CodeBuddy/...。
3. `Local Queue`：本地持久队列（断网缓存）。
4. `Transport`：gRPC/HTTPS 双协议上报。
5. `Policy Agent`：接收服务端策略（采集范围、频率、脱敏规则）。
6. `Identity Agent`：设备证书、主机绑定、租户绑定。
7. `Updater`：签名校验的自动升级（灰度）。

### Agent 关键能力

1. 支持静默安装与批量部署（Ansible/SCCM/Jamf）。
2. 支持离线缓存与断点续传。
3. 支持最小权限运行（只读目录访问）。
4. 支持远程配置下发与回滚。
5. 支持三平台安装包：Linux（deb/rpm/tar）、macOS（pkg/homebrew）、Windows（msi/zip）。

## 4.2 Server（平台服务端）

### 接入链路

1. `Ingestion Gateway`：接收 Agent Push + SSH Pull 数据。
2. `Dedup Engine`：幂等去重（event fingerprint）。
3. `Normalizer`：标准化 + 数据质量评估。
4. `Storage Router`：热数据入库，冷数据入归档存储（对象存储或本机文件系统）。

### 服务拆分建议

1. `identity-service`
2. `source-service`
3. `ingestion-service`
4. `analytics-service`
5. `governance-service`
6. `integration-service`
7. `audit-service`

### Server 平台要求

1. 支持 Linux/macOS/Windows 三平台部署（容器优先，裸机可选）。
2. 提供三平台启动脚本、服务守护配置与日志路径规范。
3. CI 必须覆盖三平台构建与基本冒烟测试。

## 4.3 三平台兼容性基线（Server + Client）

1. 目标平台：`linux/amd64`、`linux/arm64`、`darwin/arm64`、`windows/amd64`（可扩展）。
2. 文件路径与权限抽象：统一 Path Adapter，避免平台分支散落。
3. 进程管理抽象：systemd / launchd / Windows Service 统一封装。
4. 升级策略：平台差分包 + 回滚点 + 签名校验。
5. 诊断命令：三平台统一 `agentledger doctor` 输出。

## 5. 身份识别绑定体系

## 5.1 资源模型

1. `Tenant`（租户）
2. `Organization`（组织）
3. `Workspace/Project`（项目）
4. `User`（用户）
5. `Device/Host`（设备）
6. `AgentInstance`（Agent 实例）
7. `SourceBinding`（数据源绑定）

## 5.2 绑定关系

1. 用户可绑定多个设备。
2. 设备可绑定多个来源（CLI/IDE）。
3. 会话数据必须带 `tenant_id/org_id/user_id/device_id/source_id`。
4. 未绑定身份的数据进入隔离区（待认领/待审批）。

## 5.3 权限模型

1. 平台角色：`PlatformAdmin` / `OrgAdmin` / `SecurityAuditor`。
2. 业务角色：`Owner` / `Maintainer` / `Member` / `ReadOnly`。
3. 细粒度策略：按项目、来源、数据级别控制导出/查看。

## 6. 账号体系与 OAuth/SSO

## 6.1 登录能力

1. 本地账号（邮箱/密码，MFA 可选）。
2. OAuth2/OIDC 登录（Google/Microsoft/GitHub 等）。
3. 企业 SSO（OIDC/SAML，接入企业 IdP）。

## 6.2 会话与安全

1. JWT + Refresh Token。
2. 风险登录检测（IP/设备异常）。
3. 强制 MFA（可按策略开启）。
4. SCIM（后续）：自动用户/组织同步。

## 7. 办公平台接入（企微/钉钉/飞书）

## 7.1 接入方式

1. 机器人（Bot）消息推送。
2. 交互卡片（预算告警、审批、日报）。
3. Webhook 回调（告警确认、工单联动）。

## 7.2 典型场景

1. 预算阈值告警推送到企微/钉钉/飞书群。
2. 管理员在卡片内确认“提高预算/保持限制”。
3. 每日/每周 AI 使用摘要自动推送。

## 8. 归档存储（对象存储/本机文件）与多云协议支持

## 8.1 存储分层

1. 热数据：PostgreSQL/ClickHouse（查询与统计）。
2. 温数据：压缩归档对象或归档文件（按天分区）。
3. 冷数据：长期归档对象或长期归档文件（合规保留）。

### 8.1.1 存储模式

1. `object` 模式：仅对象存储（S3/OSS/COS/OBS/GCS/Azure Blob）。
2. `local` 模式：仅服务器本机文件系统（无 OSS 依赖）。
3. `hybrid` 模式：近期归档本地，长期归档对象存储。

## 8.2 对象存储抽象层（Object Storage Gateway）

统一接口：

1. `putObject/getObject/listObjects/deleteObject`
2. `multipart upload`
3. `server-side encryption`
4. `lifecycle policies`

支持目标：

1. AWS S3（S3 API / SigV4）
2. Google Cloud Storage（GCS API）
3. Azure Blob Storage（Blob REST）
4. Alibaba Cloud OSS（OSS/S3 兼容模式）
5. Tencent Cloud COS（COS/S3 兼容模式）
6. Huawei Cloud OBS（OBS/S3 兼容模式）
7. Oracle OCI Object Storage
8. MinIO/Ceph（私有对象存储）

## 8.3 本机文件存储后端（Local Filesystem Backend）

目录建议：

1. `/data/agentledger/archive/raw/YYYY/MM/DD/`
2. `/data/agentledger/archive/parquet/YYYY/MM/DD/`
3. `/data/agentledger/archive/index/`

实现要求：

1. 原子写入（临时文件 + rename）。
2. 分片滚动（按大小/时间切分）。
3. 校验和（SHA256）与清单索引。
4. 生命周期清理任务（按保留策略删除）。
5. 可选本地加密（AES-GCM）与磁盘配额告警。

## 8.4 数据格式建议

1. 原始事件归档：`JSONL + ZSTD`。
2. 分析归档：`Parquet`（按日期/租户/来源分区）。
3. 元数据索引：数据库中维护对象键或文件路径与哈希。

## 9. 企业级管控能力（必须具备）

## 9.1 安全

1. 端到端 TLS，Agent 到服务端 mTLS（推荐）。
2. KMS 管理密钥，敏感字段加密存储。
3. Token/密钥轮换与最小权限。
4. DLP 脱敏策略（PII/密钥/凭据）。

## 9.2 审计

1. 登录审计。
2. 数据访问审计。
3. 导出审计。
4. 策略变更审计。
5. Agent 操作审计（安装/升级/卸载）。

## 9.3 治理

1. 配额治理：组织/项目/用户配额。
2. 预算治理：阈值、冻结、审批解禁。
3. 客户端治理：允许/禁止客户端白名单。
4. 主机治理：在线状态、风险等级、合规状态。

## 9.4 运维

1. 多集群高可用。
2. 灰度发布与回滚。
3. 全链路监控（Metrics/Logs/Traces）。
4. 灾备与跨区域备份。

## 10. 数据保留与生命周期

1. 默认保留策略：
   - 原始事件：90 天热存 + 365 天归档存储（对象存储或本机文件）。
   - 统计聚合：长期保留。
2. 可配置策略：按租户/组织自定义。
3. 支持“一键清除”与合规删除流程。

## 11. 部署形态建议

## 11.1 私有化单集群（起步）

1. Nginx/API Gateway
2. Core Services（容器化）
3. PostgreSQL + Redis
4. Archive Storage（二选一）：
   - Object Storage（云上或 MinIO）
   - Local Filesystem（服务器本机磁盘/NAS 挂载目录）

三平台部署建议：

1. Linux：生产默认平台（容器化优先）。
2. macOS：开发/测试与小规模内部部署。
3. Windows：企业内网场景（Windows Server + MSI 服务安装）。

## 11.2 企业多环境

1. Dev/Staging/Prod 三环境。
2. 统一 IaC（Terraform）部署。
3. Secret Manager（Vault/KMS）管理。

## 11.3 SaaS 多租户（后续）

1. 逻辑租户隔离 + 行级权限。
2. 大客户可切换专有存储桶。
3. 租户级审计与计费。

## 12. 分阶段落地路线

## Phase A（MVP+）

1. 保持 SSH 模式。
2. 新增 Agent Push 基础版。
3. 完成身份绑定基础模型。
4. 完成 GitHub 风格热力图。

## Phase B（企业版）

1. OAuth/OIDC/SSO 接入。
2. 企微/钉钉/飞书告警机器人。
3. 归档存储抽象层（对象存储多云网关 + 本机文件后端）。
4. 预算与审批流。

## Phase C（成熟版）

1. 策略中心 + DLP + 合规审计。
2. 多租户 SaaS 形态。
3. 高级异常检测与成本预测。

## 13. 最终能力检查清单（上线前）

1. 双采集模式可用（SSH + Agent）。
2. P0 客户端接入完成（含国产客户端）。
3. 账号体系 + OAuth 登录可用。
4. 办公平台告警通路可用。
5. 归档存储至少支持一种本机模式与一种对象存储模式（对象存储场景下需适配 >= 3 家云厂商之一）。
6. 审计、预算、配额、权限四项治理能力上线。
7. 热力图、趋势图、会话下钻链路完整可用。
8. Server 与 Client 均通过 Linux/macOS/Windows 三平台安装与冒烟验证。

## 14. 后续增强能力（老板确认）

1. 多区域与数据主权策略。
2. Prompt/规则资产中心。
3. MCP 工具治理。
4. 质量评估体系。
5. 会话回放与复现实验室。
6. 告警编排中心。
7. 合规取证包。
8. 平台开放能力（OpenAPI/Webhook/SDK）。
