# AgentLedger 剩余缺口清单

- 文档版本：v1.0
- 更新时间：2026-03-07
- 目标：只保留“本轮全量复核后仍然明确存在”的缺口，不重复记录已完成能力。

## 1. 缺实现

### 1.1 企业治理深化

1. Rule Hub 双人审批
   - 当前已有资产/版本/发布/回滚/审批/审计闭环
   - 仍缺双人审批流程

2. Rule Hub 版本 diff
   - 当前已支持版本管理和回滚
   - 仍缺正式的版本差异比对能力

3. 告警编排中心的工单系统联动
   - 当前有规则、模拟、执行日志、外部分发闭环
   - 仍缺工单系统联动实现

4. 告警编排中心的 SLA 升级执行链
   - 当前文档有目标描述
   - 但未见明确升级执行链与回归证据

5. 数据主权的 KMS / 区域归档映射
   - 当前有 policy / region mappings / replication jobs
   - 仍缺区域级密钥管理映射与区域归档策略落地

6. Agent 生命周期审计
   - 平台侧审计已完整
   - 但未见 Agent 安装/升级/卸载审计入口

7. SCIM / SAML / 强制 MFA / 风险登录
   - 当前已有 provider discovery、OAuth/OIDC、外部断言登录
   - 更深企业 IAM 集成仍未落地

8. DLP / Legal Hold
   - 当前有 evidence bundle 与审计导出
   - 但未见 DLP 与 Legal Hold 代码入口

### 1.2 企业 C/S 深化

1. Store-and-Forward 本地持久队列
2. Policy Agent / 远程配置下发
3. Agent Updater / 自动升级
4. 静默安装与批量部署资产

这些项在 `docs/10` 中属于企业 C/S 完整体，目前未见明确实现入口。

## 2. 缺验证

1. Agent Push 的 gRPC mTLS 端到端链路
   - 当前客户端与 ingestion-gateway 都有 TLS/mTLS 单元测试
   - 但缺专门的端到端 smoke / E2E 证明整链可用

2. 某些长期蓝图项只有文档，没有代码，也就不存在验证入口
   - 这类项不应误记成“验证失败”
   - 应继续维持为“仅规划/长期蓝图”

## 3. 缺文档

1. 需要把本轮复核后的事实分层长期保留
   - `docs/00`：项目定位与文档判定顺序
   - `docs/01`：基础需求适用边界
   - `docs/11`：企业规划来源与“规划 vs 当前实现”核对结论
   - `docs/16`：功能核对矩阵
   - `docs/17`：回归验证执行记录

2. 如后续继续推进企业能力深化，建议补一份新的 `docs/releases/*`
   - 专门记录“全量规划复核与回归验证”这一轮结论
   - 避免后续交接继续依赖历史聊天记录

## 4. 当前不应再重复开发的内容

以下内容本轮已确认“已实现且已验证”，不应再被误判成待开发：

1. 数据主权首版（policy / mappings / replications）
2. Rule Hub 首版闭环
3. MCP 工具治理首版闭环
4. 告警编排中心真实分发闭环
5. 合规取证包
6. OpenAPI / Webhook / API Key / 8 语言 SDK
7. quality v2 基础版
8. replay v2 基础版
9. integration callback / routing / downstream smoke
10. Agent 多架构构建、FR-505、CI / Release 门禁链
