# AgentLedger UI/UX 设计规范

- 文档版本：v1.0
- 更新时间：2026-03-01
- 关联文档：`01-功能需求规格说明.md`

## 1. 设计目标

1. 让“会话检索 + token/cost 分析”在一个工作流里完成。
2. 让跨主机来源的可追溯性一眼可见。
3. 控制复杂度：高级能力强，但基础操作 3 次点击内完成。

## 2. 视觉概念（V1）

**设计方向：`Control Tower`（控制塔仪表风）**

1. 气质：专业、稳定、可审计。
2. 特征：高信息密度 + 强分层 + 高可读数据视觉。
3. 记忆点：顶部“来源雷达条” + 中央“GitHub 风格使用热力图” + 右侧“限额警戒线”。

## 5. 核心用户流程

## 5.1 首次接入流程

1. 进入 `Sources`。
2. 添加本地目录并校验。
3. 添加 SSH 主机并测试连接。
4. 返回 `Dashboard` 查看首次聚合结果。

## 5.2 异常成本排查流程

1. 在 Dashboard 看到日峰值异常。
2. 点击下钻到 `Analytics > Daily`。
3. 过滤来源与模型。
4. 跳转 `Sessions` 查看具体会话消息与 token 分解。

## 5.3 远程问题复盘流程

1. 在 `Sessions` 选择远程来源。
2. 输入关键词并选择访问模式（实时 / 缓存 / 混合）。
3. 查看命中片段与来源主机、数据新鲜度标签。
4. 打开会话详情并导出审计片段。

## 6. 关键页面规范

## 6.1 Dashboard

模块：

1. KPI 条：`总 token`、`总成本`、`活跃会话`、`异常来源`。
2. 成本趋势图：按日折线 + 阈值带。
3. GitHub 风格日历热力图：按日期展示 AI 使用强度。
4. 来源健康雷达：连接成功率/延迟/失败率。
5. 模型占比环图：支持点击联动过滤。

热力图交互要求：

1. 指标切换：`tokens` / `cost` / `sessions`。
2. 过滤联动：按客户端类型（CLI/IDE）、工具、主机、模型过滤。
3. Tooltip：显示日期、总量、Top 工具、同比信息。
4. 点击某天：下钻到 `Sessions` 列表并自动带入日期过滤。
5. 新鲜度标识：实时数据与缓存数据颜色边框区分。

## 6.2 Sessions

模块：

1. 左侧过滤器：来源、工具、模型、时间。
2. 中部列表：会话标题、来源、时间、token、cost。
3. 右侧详情：消息时间线 + usage 轨迹 + 来源追溯。

交互：

1. `Cmd/Ctrl + K` 全局搜索。
2. 键盘 `J/K` 上下切换会话。
3. `Shift + Enter` 在详情内快速复制片段。

## 6.3 Sources

模块：

1. 来源卡片：主机、路径、状态、上次成功。
2. 模式配置：`realtime` / `sync` / `hybrid` 开关与同步周期。
3. 读取诊断：实时拉取耗时、同步任务状态、缓存新鲜度。
4. 安全标识：指纹校验状态、只读策略状态。

## 6.4 Budgets

模块：

1. 预算总览：月预算、已用、剩余、预测超支日。
2. 阈值线：50/80/100% 可配置。
3. 告警记录：时间、规则、命中对象。

## 8. 动效规范

1. 首屏加载：卡片分层渐入（80ms stagger）。
2. 数据刷新：图表平滑过渡，避免闪屏。
3. 异常告警：阈值线轻脉冲，不使用高干扰动画。

## 9. 可用性与可访问性

1. 键盘可达：所有核心功能支持键盘操作。
2. 对比度：正文与背景对比度 >= WCAG AA。
3. 响应式：
   - `>=1280px` 三栏布局
   - `768~1279px` 双栏布局
   - `<768px` 单栏折叠（移动只读优先）

## 10. 文案策略

1. 所有成本展示明确“原始/估算”标签。
2. 远程数据展示统一标记“实时/缓存/混合”与缓存时间戳。
3. 热力图颜色等级提供图例与文本说明，避免误读。
4. 错误提示包含“可执行下一步”。

## 11. 设计验收标准

1. 新用户 10 分钟内可完成来源接入与首次查询。
2. 异常排查路径（峰值 -> 会话）不超过 4 步。
3. 关键页面可在 1366x768 与 390x844 正常使用。
4. 用户可区分本地与远程数据来源，不产生误解。
